## Conventions

- ALWAYS use `[[wikilinks]]` for internal links.
- Source callout pattern: `> [!source]- Source` followed by the canonical MA help URL.
- Section nav footer pattern: `тЖР [[prev]] | [[next]] тЖТ` then `Part of [[Section]] ┬╖ [[000 Map of Content]]`.
- Do NOT create links to notes that don't exist unless asked.
- Keyword notes use backtick code format for inline CLI syntax: `` `Store Preset N /global /overwrite /noconfirm` ``

## Frontmatter fields

### All notes
- **type**: `section` | `page` | `keyword` | `key` | `quick-start` | `moc`
- **slug**: the original MA help slug, e.g. `key_presets_create`
- **url**: canonical source URL
- **tags**: array тАФ `type/{type}`, `section/{slug}` for non-keyword nodes
- **aliases**: from `meta-keywords` and `meta-searchwords` in page `<head>`
- **cssclasses**: `gma2-{type}` тАФ drives per-type reading-view styling via the
  `.obsidian/snippets/note-types.css` snippet

### `page` notes (additional)
- **section**: parent section title string
- **ma2_section**: parent section slug
- **section_ref**: wikilink to parent section note
- **prev_page**: wikilink or null
- **next_page**: wikilink or null
- **depth**: `2` (direct child) or `3` (grandchild within section)

### `keyword` notes (additional)
- **keyword**: the exact keyword string, e.g. `Store`
- **keyword_type**: `function` | `object` | `helping` | `special-char`
- **related_key**: wikilink to corresponding console key note (if exists), e.g. `[[Keys/Store]]`

### `key` notes (additional)
- **key_label**: display label of the key, e.g. `Store`
- **related_keyword**: wikilink to corresponding keyword note (if exists)

### `section` notes (additional)
- **page_count**: number of direct child pages
- **pages**: array of wikilinks to all pages in this section

## grandPA2-buddy RAG integration

These notes are the primary knowledge source for the `ma2-onPC-MCP` RAG pipeline.

**Priority tiers for embedding/retrieval:**
1. **Tier 1 (embed always):** `Keywords/` тАФ all CLI keyword notes
2. **Tier 2 (embed always):** `Pages/Command Syntax and Keywords/`, `Pages/Presets/`, `Pages/Cues and Sequences/`, `Pages/Executors/`, `Pages/Patching/`
3. **Tier 3 (embed on demand):** All remaining `Pages/` and `Sections/`
4. **Tier 4 (skip):** `Keys/` (physical hardware, low RAG value)

**Known gotchas captured in notes:**
- `Fixture` shortcut = `F`, NOT `Fix` (Fix = freeze/lock)
- `/global` flag is case-sensitive тАФ not `/Global`, not `/g`
- `List Preset` "Special=Normal" does NOT indicate Selective after `/global` store
- `List Preset N.1 Thru N.9999` enumerates pool тАФ bare `list preset N` mis-scopes
- `SaveShow` requires `/noconfirm` in automation contexts

## Analysis tooling (`scripts/`)

MA2-domain refactor of the `linkedin-learning-extractor` analytics/graph stack
(no-analog features тАФ instructors, skills, durations, video timestamps тАФ dropped).
All read the vault via `lib/vault.js` (`loadVault`, `stripBoilerplate`) and run
in-memory; output goes to the **gitignored** `analytics/`.

- `semantics.js` (`npm run semantics`) тАФ MA2 command syntax: prompt examples
  (`[Channel]> тАж`), keyword-led inline-code commands, `/flags`, keyword usage.
- `analytics.js` (`npm run analytics <summary|topics|similar <note>|orphans>`) тАФ
  counts/link-density/orphans, pages-per-section, Jaccard similarity (MOC links
  excluded so overlap is meaningful).
- `nlp.js` (`npm run topics`) тАФ local term+keyword topics; optional `openai`
  provider via fetch (`NLP_PROVIDER=openai`, no SDK).
- `viz.js` (`npm run viz`) тАФ standalone `analytics/graph.html` (vis-network CDN).
- `db.js` (`npm run db <import|summary|search <q>|similar <note>|orphans>`) тАФ SQLite
  catalog (`better-sqlite3`) in `analytics/vault.db` with an **FTS5** full-text index.
- `graph.js` (`npm run graph <search|similar|index|serve|cypher>`) тАФ in-memory search +
  HTTP API (:3100). **Optional Neo4j** (`neo4j-driver`, gated on `NEO4J_URI`/
  `NEO4J_PASSWORD`): `index` loads the graph, `search` queries its full-text index;
  `cypher` exports a loadable script. See `docker-compose.yml`.

Dev tooling: ESLint + Prettier (vault markdown excluded via `.prettierignore`),
Husky (pre-commit lint-staged, commit-msg commitlint, pre-push test+audit), all
also enforced by `ci.yml`.

### Build notes / gotchas

- **Quartz is pinned to `v4.5.2`** in `.github/workflows/sync.yml` (gh-pages job). Quartz **v5**
  dropped the clone-and-build model this workflow uses (`quartz.config.ts` / `.quartz/plugins`
  are now scaffolded by `npx quartz create`), so a bare clone of `main` breaks `npx quartz build`.
  Do **not** un-pin without reworking the job to the `npx quartz create` flow. The workflow runs
  on **Node 22** (Quartz's engine requirement).
- **`github-browse` link conversion** (`scripts/convert-links.js`) resolves wikilinks by
  vault-root-relative path, then case-insensitively. Notes use **path-qualified** links
  (`[[Sections/Networking]]`, `[[Pages/.../Note]]`); bare links are only the root `[[000 тАж]]`
  index files.
- **`npm run audit`** (`scripts/audit.js`) is a read-only conformance check against this SOP:
  it reports unresolved wikilinks, structure/`type`-vs-location mismatches, missing required
  frontmatter, and missing source-callout / nav-footer conventions. Run it after any extract.
  Current state: **PASS** (0 unresolved links, 0 structure/frontmatter issues).
- **`npm test`** (`scripts/test/`, built-in `node --test`, no extra deps) covers the
  link-conversion, slug/filename, and audit logic.
- **CI** (`.github/workflows/ci.yml`) runs `npm test` + `npm run audit` on every push to `main`
  and on PRs тАФ the quality gate that keeps broken links / frontmatter off `main`.
- **`npm run stats` / `npm run moc`** write live note counts into `README.md` and
  `000 Map of Content.md` (no longer placeholder `тАФ`).
